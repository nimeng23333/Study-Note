1. ### Cookie 有效期设置
    

登录时，token 会被保存到 cookie，有效期约 1 年（受浏览器限制）：

index.tsxLines 220-223

```TypeScript
setCookieLongTerm(USER_TOKEN_COOKIE_KEY, token, {
  secure: true, // 仅通过 HTTPS 传输
  sameSite: 'none',
});
```

2. ### 但登录状态不一定会保持一年
    

原因：

#### Token 验证机制

每次页面加载时，都会验证 token 是否有效：

_app.page.tsxLines 317-323

```TypeScript
const userinfo = await userService.getUserInfoByToken(usertoken);
    if (!userinfo) {
  deleteCookie(USER_TOKEN_COOKIE_KEY, appContext.ctx);
  deleteCookie(config.platformTokenCookieKey, appContext.ctx);
  return null;
}
```

  

如果后端 token 已过期或无效，即使 cookie 还在，也会删除 cookie 并要求重新登录。

#### API 请求拦截

每次 API 请求返回 401 时，也会清除登录状态：

index.tsLines 90-99

```TypeScript
if (data.code === RequestEnums.OVERDUE) {
  console.log('登录失效，请重新登录');
  // 登录信息失效，应跳转到登录页面，并清空本地的token
  Cookie.remove('userToken');
  if (router.router?.route && router.router.route.indexOf('pretrain') !== -1) {
    // localStorage.setItem('showLogin', '1');
  } else {
    router.push('/');
  }
  return Promise.reject(extraData);
}
```

#### 没有 Token 刷新机制

前端中没有 token 自动刷新的逻辑，意味着：

- 如果后端 token 有效期短于 1 年（例如 30 天、90 天），即使 cookie 未过期，token 失效后也需要重新登录。
    

3. ### 退出登录会清除所有 Cookie
    

index.tsxLines 1808-1820

```TypeScript
onClick={async () => {
   await logout();
  localStorage.removeItem('token');
  deleteCookie(USER_TOKEN_COOKIE_KEY);
  deleteCookie(USER_TOKEN_COOKIE_NEW_KEY);
  deleteCookie(config.platformTokenCookieKey, { domain: '.liblib.art' });
  localStorage.removeItem('modal_countdown');
  modalHandlers.close();
  if (showOtherLogo && enterpriseInfo?.loginPath) {
    window.location.assign(enterpriseInfo?.loginPath);
  } else {
    window.location.assign(i18n.URLS.Home);
  }
}}
```

## 总结

- Cookie 有效期：约 1 年（浏览器限制）
    
- 实际登录保持时间：取决于后端 token 的有效期
    
- 如果后端 token 有效期短于 1 年，即使 cookie 未过期，token 失效后也需要重新登录
    
- 如果后端 token 有效期长于 1 年，cookie 会在 1 年后过期，需要重新登录
    

因此，不能保证一年都保持登录状态，实际时长取决于后端 token 的有效期。
